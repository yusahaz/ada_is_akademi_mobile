# Flutter AI Context

This folder stores persistent context used by Cursor to behave like an expert-level UI/UX designer and Flutter engineer.

## Files
- `architecture.md`: architectural boundaries and layering.
- `design-system.md`: design tokens and responsive UI standards.
- `components.md`: reusable component contracts and patterns.
- `workflow.md`: analysis-first implementation workflow.
- `tasks.md`: lightweight task board for ongoing work.
- `memory.md`: long-term decisions, assumptions, and risks.

## Usage
1. Read relevant files before implementation.
2. Analyze impact and risks before coding.
3. Update `tasks.md` and `memory.md` after each meaningful change.

# Components

## Reusability Standards
- Build modular, reusable widgets with clear APIs.
- Keep constructor parameters minimal and explicit.
- Prefer predictable state patterns for interactive widgets.

## Stability Rules
- Do not modify existing component contracts unless required.
- When extending a component, preserve backwards compatibility.
- If replacement is needed, provide a migration-safe path.

## Quality Checklist
- Responsive behavior validated.
- Accessibility states covered (focus, disabled, error).
- Theming and dark/light compatibility considered when relevant.
